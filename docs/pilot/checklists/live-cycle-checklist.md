# Live Cycle Checklist

Use this while the cycle is active.

## Submission phase

- [ ] Submissions opened at the planned time
- [ ] Participants can access the intended surface
- [ ] Contributions are being recorded
- [ ] Any access or save failures are logged
- [ ] No silent operator-side data edits were made

## Transition to closed submissions

- [ ] Submissions closed at the planned time
- [ ] Late submissions policy applied consistently
- [ ] Close event recorded in operator notes if delayed or manual

## Intervention routing and release

- [ ] Routing ran only after submissions closed
- [ ] Routing outputs were inspected
- [ ] Digests exist before release
- [ ] Release happened only after prerequisites were satisfied
- [ ] Any rerun or release delay was logged

## Baseline thread visibility

- [ ] Baseline visibility opened as planned
- [ ] No intervention-only artifacts were shown
- [ ] Any manual recovery action was logged

## Reflection and feedback

- [ ] Responses are being captured
- [ ] Feedback instrument is reachable
- [ ] Missing feedback or access issues are logged
- [ ] Reflection window stayed within the planned bounds

## Failure handling

- [ ] Failures were logged immediately
- [ ] Pause, recovery, or rerun decisions were explicit
- [ ] Partial data was preserved rather than overwritten
